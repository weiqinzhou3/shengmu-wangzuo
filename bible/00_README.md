## 设定优先级

发生冲突时按以下顺序处理：

1. `11_CANON_LOCK.md` 中的硬规则
2. 已经发布的正文事实
3. 本小说圣经中的正式设定
4. 分卷大纲
5. 单章计划
6. 临时灵感

不得为了制造爽点，临时破坏死亡规则、资源规则、人口规则或八阶唯一性。

## 版本管理

每次重大修改都要记录：

- 修改日期
- 修改文件
- 修改原因
- 影响章节
- 是否需要回查已发布正文

建议使用 Git 管理，并为每次正式发布的章节打标签。

## 新设定回写规则

正文中首次出现下列内容时，必须在同一提交中更新对应文件：

- 新人物：`15_CHARACTER_BIBLE.md` 与 `state/CHARACTER_STATE.md`
- 新事件：`state/EVENT_LEDGER.md`
- 新背景故事或历史：`state/BACKGROUND_LORE.md`
- 新兵种或兵种例外：`state/UNIT_BESTIARY.md`
- 新国家、派系或外交关系：`14_NATION_ALIASES.md`、`18_FACTION_DOSSIERS.md`、`state/NATION_STATE.md` 与 `state/FACTION_STATE.md`
- 新宝物或宝物状态变化：`16_ARTIFACT_SYSTEM.md` 与 `state/ARTIFACT_LEDGER.md`
- 新地点、路线或地图知识：`17_WORLD_MAP.md` 与 `state/MAP_STATE.md`
- 新爱情或关系变化：`state/RELATIONSHIP_MATRIX.md`
- 新死亡或不可逆事件：`state/DEATH_LEDGER.md`

不得只写入正文而不更新设定账本。
