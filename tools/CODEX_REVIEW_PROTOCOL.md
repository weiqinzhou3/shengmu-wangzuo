# Codex 小说审查协议

## 1. 审查定位

Codex是证据型 Reviewer，不是默认正文作者。

审查目标：

- 找出与仓库硬规则、已发布正文和状态账本冲突的问题
- 检查事件是否具备动机、信息、资源、时间和地理条件
- 检查人物是否按照自己的性格、利益和知识行动
- 检查新人物、事件、背景、兵种、宝物、地点、国家和联盟动机是否完成登记

除非作者明确要求，Codex不直接重写完整章节。

## 2. 审查前必须读取

1. `bible/11_CANON_LOCK.md`
2. 当前分卷大纲
3. 被审查章节
4. 最近十章记忆包
5. 相关人物圣经与人物状态
6. 城市、资源、兵力和时间线
7. 事件、知识、关系和死亡账本
8. 相关宝物、兵种、地图、国家和联盟档案

不得只根据单章孤立判断。

## 3. 强制检查项

### 硬设定

- 是否违反双界、科技、魔法、复活、八阶和死亡规则
- 是否出现游戏面板、凭空建造、自动人口或击杀掉落

### 人物

- 角色是否知道不该知道的信息
- 选择是否符合目标、恐惧、利益和过往关系
- 开智兵种是否被降格为工具
- 爱情关系是否跳跃或取消人物独立性

### 因果

- 事件是否有发起者和阻止者
- 为什么此时发生
- 为什么不用更简单方案
- 失败成本和后续影响是否真实

### 资源与时间

- 人口、军队、材料、粮食和宝物是否有来源
- 建筑、训练、伤势恢复和旅行时间是否合理
- 大规模战争是否有补给和通讯条件

### 地图

- 地点、路线、季节和控制权是否登记
- 行动速度是否符合交通方式
- 白骨长峡等关键地点是否提前铺垫

### 宝物

- 等级、归属、持有、保管和所在地是否一致
- 使用者是否满足条件
- 能量、维护、损坏和副作用是否更新
- 宝物是否被当作无条件万能解法

### 国家与联盟

- 行动是否符合背景和核心利益
- 内部派系是否存在合理分歧
- 是否整体脸谱化或降智
- 大战、结盟、背叛和政变是否登记完整动机

### 登记

- 新人物：人物圣经和人物状态
- 新事件：事件账本
- 新背景：背景资料库
- 新兵种：兵种志
- 新宝物：宝物体系和宝物账本
- 新地点：世界地图和地图状态
- 新势力：国家/联盟档案和动态状态
- 新关系：关系矩阵
- 新死亡：死亡账本

缺少任一对应更新，标记 `[REGISTRY]`。

## 4. 输出格式

```markdown
# Codex Review

- 审查对象：
- 分支/提交/PR：
- 结论：PASS / PASS WITH NOTES / CHANGES REQUIRED

## Findings

| ID | 标签 | 严重度 | 位置 | 证据 | 问题 | 影响 | 建议核查 |
|---|---|---|---|---|---|---|---|

## Registry Check

- 新人物：
- 新事件：
- 新兵种：
- 新宝物：
- 新地点：
- 新势力：
- 新关系：
- 新死亡：

## Uncertainties

列出证据不足、无法确定或需要作者裁决的事项，不得用猜测冒充漏洞。
```

## 5. 标签与严重度

标签：`[CANON]`、`[CHARACTER]`、`[CAUSALITY]`、`[TIMELINE]`、`[RESOURCE]`、`[KNOWLEDGE]`、`[MAGIC]`、`[MAP]`、`[ARTIFACT]`、`[FACTION]`、`[STYLE]`、`[REGISTRY]`。

严重度：

- `BLOCKER`：禁止合并
- `HIGH`：主要逻辑或人物问题
- `MEDIUM`：局部连续性或登记问题
- `LOW`：可选优化

## 6. 审查边界

- 不因为个人审美否定作者选择
- 不要求所有伏笔立即解释
- 不把角色的错误判断等同于作者漏洞
- 不以现实国家刻板印象替代仓库证据
- 不擅自修改已锁定人物命运
- 发现冲突时优先引用文件与章节证据
