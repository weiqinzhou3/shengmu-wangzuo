# Codex 审查记录

> Codex依据仓库正文和设定提供证据型审查。作者负责判断和修订，读者负责体验与方向裁决。

## 状态定义

- `OPEN`：尚未处理
- `ACCEPTED`：作者接受并准备修改
- `RESOLVED`：已通过Codex复审确认
- `REJECTED`：作者保留原文并说明理由
- `DEFERRED`：确认存在，但延后到指定阶段处理
- `PENDING_REVIEW`：作者已修订，等待Codex复审确认
- `PARTIAL`：核心方向已修订，但仍有关键条件或登记未闭合

## 严重度

- `BLOCKER`：违反硬规则或造成剧情无法成立，禁止合并
- `HIGH`：主要人物动机、因果、资源、时间或法权存在明显漏洞
- `MEDIUM`：局部连续性、知识边界或登记遗漏
- `LOW`：措辞、轻微重复或可选优化
